# 他エージェントへの委譲（dev send / dev peek）

このマシンには `dev` コマンドがあり、同じ tmux セッションの隣ペインで動く
AI エージェントとやり取りできる。

- `dev send <agent> -` : 標準入力の内容をエージェントの入力欄へ送信（複数行対応）
- `dev peek <agent> [-n N]` : エージェントのペインの直近出力を読む（既定 120 行）
- `<agent>` は `codex`（実装・作業者）または `agy`（第三者レビュー）

## 使い分け

- **codex**: 実装・修正・テスト追加などの実作業を任せる
- **agy**: レビュー・別視点の検証を求める。agy がない環境では `dev send agy` が
  エラーになるので、その場合は「この環境に agy はありません」とユーザーに伝える

## 手順

### 1. 送信前の状態確認

`dev peek <agent> -n 30` を実行し、対象 CLI が起動していて応答待ちでないかを見る。
`dev send` が「シェルに戻っています」エラーを返したら、ユーザーに
「ペインで <agent> を起動してください」と伝えて中断する（勝手に起動しない）。

### 2. 依頼を送る

必ずヒアドキュメントで送る（クォート事故防止・複数行対応）:

```bash
dev send codex - <<'EOF'
[FROM: Claude Code] 以下のタスクをお願いします。

## タスク
（1段落で要約）

## 対象
（ファイルパス・関数名など具体的に）

## 制約
（触ってはいけない範囲、テスト要件、コミット禁止など）

完了したら最終行に「DEV_DONE」、続行不能なら「DEV_BLOCKED: 理由」とだけ出力してください。
EOF
```

- 先頭の `[FROM: Claude Code]` は受け側のグローバル指示で定義済みのプロトコル。省略しない
- 依頼は自己完結させる。受け側はこの会話を見ていないので、文脈・パス・制約をすべて含める

### 3. 完了を待つ

送信後 60 秒待ってから `dev peek <agent> -n 40` で確認する。
末尾付近に `DEV_DONE` / `DEV_BLOCKED` が現れるまで「60秒待機 → peek」を繰り返す（上限 10 回）。

- 上限に達したら、最後の peek 結果を要約してユーザーに報告し、指示を仰ぐ
- peek の出力を全文貼らず、要点だけ要約する

## 人間が最終判断する（最重要）

- 委譲結果を**勝手に承認・コミット・マージしない**。完了を検知したら、変更内容・結果の
  要約をユーザーに提示し、判断を仰ぐ
- codex の成果に問題を見つけても、修正の再依頼はユーザーの同意を得てから
- agy のレビュー結果は「第三者意見」として提示するのみで、採否はユーザーが決める

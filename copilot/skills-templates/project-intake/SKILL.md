# Project Intake

## 目的

実装や設計に入る前に、ゴール・制約・未確定事項を短時間で揃える。

## 使う場面

- 新しい変更依頼を受けた直後
- 要件が曖昧で、まず整理が必要なとき
- 複数 agent 間の handoff を明確にしたいとき

## 確認項目

1. 何を達成したいか
2. 何を今回のスコープ外とするか
3. 既知の制約は何か
4. まだ確認できていない前提は何か
5. 検証方法は何か

## 出力テンプレ

1. Goal
2. Non-Goals
3. Facts
4. Unknowns
5. Validation Plan

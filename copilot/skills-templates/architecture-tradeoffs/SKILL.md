# Architecture Tradeoffs

## 目的

設計判断の理由を残し、代替案との比較を明確にする。

## 確認項目

1. 既存の責務分割はどうなっているか
2. 今回の変更境界はどこか
3. 選択肢ごとの利点と欠点は何か
4. 移行やロールバックは必要か
5. テスト観点は何か

## 出力テンプレ

1. Current State
2. Options
3. Decision
4. Impact
5. Migration
6. Test Strategy
